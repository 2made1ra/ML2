# Изменения общего слоя данных в `lab_3.ipynb` (строка ~159)

Пока внесены только правки в **общий слой данных**, от которого зависят **обе модели**.

## Что изменено

- **`max_samples`**: верхний лимит streaming-кэша поднят с `5_000` до `25_000`.
- **`cache_image_size`**: поднят до `384`. **Старый кэш не пересоздаётся автоматически** — при необходимости его нужно пересобрать вручную.
- **`WeightedRandomSampler`** для train-loader: редкие классы **чаще попадают в батчи**.
- **`use_class_weights=False`**: веса классов в loss не дублируют эффект сэмплера; комбинация *sampler + class weights* обычно **переусиливает** редкие классы.
- **Оценка RAM** перед созданием `DataLoader`: при текущих `batch_size=192`, `224×224`, `num_workers=0` получается **порядка ~110 MB** на prefetched тензоры. Изображения остаются **JPEG на диске**, весь датасет в ОЗУ **не загружается**.
- **Предупреждение в коде**, если старый кэш был собран с **меньшим** `cache_image_size`.
- **JSON ноутбука** проверен на корректный парсинг. **Обучение не запускалось**. Python из окружения в той оболочке **недоступен/заблокирован** — **AST-проверка** к ячейкам **не выполнялась**.

## Датасет huggan/wikiart и `max_samples`

На странице [huggan/wikiart](https://huggingface.co/datasets/huggan/wikiart) фактический размер split в streaming/viewer-режиме может оказаться **меньше 25_000**. Код трактует `25_000` как **верхний лимит**: если записей меньше, кэш остановится на **фактически доступном** количестве.
