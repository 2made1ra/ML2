# Рекомендации по приближению к `val_acc ≈ 0.97` (FashionMNIST, `StrongerCNN`)

Контекст после последних запусков в ноутбуке:

- эксперимент `stronger_cnn_pipeline` дал около **`best_val_acc = 0.9503`**, `test_acc = 0.9478`;
- финальный прогон этой конфигурации показал **`Final best val acc = 0.9508`**, `Final best test acc = 0.9484`;
- при этом в логах финального обучения уже встречался пик **`val_acc = 0.9518`**, но итоговая модель сохранялась по `monitor_metric="val_loss"`.

Главный вывод: текущая архитектура уже достаточно сильная, и ближайший путь к `0.97` лежит не через резкое усложнение сети, а через **точную настройку pipeline под максимум `val_acc`**.

## 1. Что сейчас мешает добрать точность

По текущим результатам есть три вероятных ограничения:

1. Модель выбирается по `val_loss`, а не по `val_acc`, поэтому реальный пик валидационной точности может теряться.
2. Для FashionMNIST текущая регуляризация выглядит немного жёсткой:
   - `label_smoothing = 0.1`;
   - `weight_decay = 7e-4`;
   - `dropout = 0.15`;
   - `RandomHorizontalFlip` и `RandomErasing`.
3. Сильный pipeline уже дошёл до `~0.95`, значит следующий прирост вероятнее даст **аккуратное снятие лишнего шума**, а не ещё одна тяжёлая архитектура.

## 2. Наиболее перспективная следующая модель

Именно под цель `val_acc >= 0.97` логично пробовать следующий accuracy-oriented вариант:

- та же архитектура `StrongerCNN(channels=(64, 128, 256, 256))`;
- `optimizer = AdamW`;
- `scheduler = OneCycleLR`;
- **мониторинг `val_acc`**;
- **более мягкая аугментация**:
  - оставить `RandomCrop(28, padding=2)`;
  - оставить лёгкий `RandomAffine`;
  - убрать `RandomHorizontalFlip`;
  - убрать `RandomErasing`;
- ослабить регуляризацию:
  - `label_smoothing = 0.03`;
  - `weight_decay = 3e-4`;
  - `dropout = 0.1`;
- увеличить длительность обучения:
  - `epochs = 60`;
  - `early_stopping_patience = 10`.

Это не гарантирует `0.97`, но по последним данным это самый разумный следующий шаг с высоким шансом прироста.

## 3. Почему именно такой вектор лучше, чем ещё сильнее усложнять сеть

- Архитектурный запас уже есть: residual-модель заметно лучше простых CNN.
- На FashionMNIST слишком агрессивная регуляризация часто ухудшает именно accuracy, даже если выглядит полезной по loss.
- Логи финального обучения показывают, что модель уже умеет доходить до более высоких значений `val_acc`, но текущий критерий выбора чекпоинта и избыточный шум в train pipeline могут не давать их закрепить.

## 4. Что добавлено в ноутбук

В `LAB_II/notebooks/solution.ipynb` стоит держать отдельный эксперимент наподобие:

- `exp_name = "stronger_cnn_precision_tuned"`;
- transform с `RandomCrop + лёгкий RandomAffine`;
- без `RandomHorizontalFlip` и `RandomErasing`;
- `monitor_metric = "val_acc"`;
- уменьшенные `label_smoothing`, `weight_decay`, `dropout`;
- более длинное обучение и более терпеливая early stopping.

Именно этот эксперимент должен быть первым кандидатом на роль нового `best_exp`.

## 5. Если после этого всё ещё нет `0.97`

Тогда следующий узкий перебор стоит делать так:

1. `label_smoothing`: `0.03 -> 0.0`.
2. `dropout`: `0.1 -> 0.0`.
3. `weight_decay`: `3e-4 -> 1e-4`.
4. `scheduler`: сравнить `OneCycleLR` и `cosine_warmup` при той же длине обучения.
5. Повторить 2-3 запуска с разными `seed`, потому что для целевой границы `0.97` разброс уже становится заметным.

## 6. Практическое ожидание

Реалистичная цель ближайшего шага:

- сначала стабильно поднять `best_val_acc` в область **`0.955-0.962`**;
- затем уже добивать оставшийся зазор узкой настройкой regularization и scheduler.

Прыжок с `0.9508` сразу в `0.97` возможен, но скорее маловероятен без удачного сочетания seed, чекпоинтинга и очень аккуратной конфигурации обучения.
