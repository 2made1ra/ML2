2\. Выбор задачи и данных
-------------------------

1.  Выберите набор данных для задачи **классификации** (целевой признак принимает конечное число значений) или **регрессии** (формально бесконечное число значений).​
    

Рекомендуется:

*   использовать датасет, с которым вы уже работали в прошлом семестре и получали хорошие результаты;
    
*   либо взять данные из открытых источников: Kaggle, OpenML, соревнования прошлых семестров.Допускается использовать датасеты из текущего семестра (иногда интерпретация признаков может быть спорной, но это не критично).
    

3\. Предварительный анализ данных (**до 2 баллов**)
---------------------------------------------------

1.  Выполните первичный анализ данных: опишите, какие признаки _на первый взгляд_ могут быть полезны для предсказания. Допускается «около‑статистический» анализ (простые описательные статистики, группировки, базовые визуализации).​
    

Обязательные элементы зависят от типа задачи:

*   **Если задача классификации**:
    
    *   проанализируйте результаты группировки по целевой переменной;
        
    *   сравните статистические показатели признаков для разных классов;
        
    *   сделайте хотя бы один график из семейства catplot (seaborn), где hue — целевая переменная.​
        
*   **Если задача регрессии**:
    
    *   выполните корреляционный анализ числовых признаков с целевой переменной;​
        
    *   категориальные признаки проанализируйте через группировки: группировка по отдельному категориальному признаку и сравнение статистик целевой переменной по группам;
        
    *   постройте хотя бы один «скаттерограммо‑подобный» график, где по одной из осей (обычно по оси y) отложена целевая переменная.​
        

Во всех случаях ожидается также базовый визуальный анализ (распределения, box-plot, violin plot и т.п.).​

4\. Обучение моделей (до 2 баллов за модель)
--------------------------------------------

1.  Выберите **две разные** модели машинного обучения Желательно существенно отличающиеся по природе, например: дерево решений и логистическая регрессия; бустинг и линейная модель; RandomForest и SVM, а не только LightGBM и CatBoost.​
    
2.  Качественно обучите каждую модель: подберите разумные гиперпараметры, разделите данные на train/validation, при необходимости используйте кросс‑валидацию.​
    

Для каждой модели:

*   оцените метрики качества и поведение предсказаний;
    
*   проанализируйте, на каких объектах модель ошибается сильнее всего;
    
*   сравните, совпадают ли объекты, на которых разные модели ошибаются.​
    

5\. Глобальная значимость признаков (классические методы, до **3 баллов** за модель)
------------------------------------------------------------------------------------

1.  Оцените **глобальную** важность признаков для каждой модели «родными» для scikit-learn методами:
    

*   используйте permutation\_importance для оценки важности признаков;
    
*   постройте графики partial dependence (PDP) для интерпретации влияния признаков на целевую переменную.
    

Рекомендации:

*   ответьте на вопрос: обращают ли разные модели внимание на одни и те же признаки или на разные;
    
*   опционально сравните **feature importance / веса признаков** (если использовались деревья решений, бустинги или линейные модели) с результатами permutation importance;​
    
*   partial dependence лучше строить не для всех признаков, а для условно топ‑5/10 по важности, так как вычисления могут быть длительными;​
    
*   обязательно добавьте текстовые комментарии к полученным графикам и выводам.
    

6\. Глобальная значимость признаков (SHAP, до **3 баллов** за модель)
---------------------------------------------------------------------

1.  Оцените **глобальную** значимость признаков с помощью «модных» библиотек интерпретации, в первую очередь — shap.​
    

Обязательно:

*   используйте summary\_plot / beeswarm‑график для глобальной важности;
    
*   сравните важные признаки по SHAP с результатами «классических» методов (permutation importance, PDP, стандартные feature importance).​
    

Практические советы:

*   вычисления SHAP могут быть долгими, поэтому используйте не всю выборку, а подмножество (например, часть валидационной выборки);​
    
*   постарайтесь прокомментировать наиболее значимые признаки и их связь с изменением целевой переменной (как рост/уменьшение признака влияет на предсказание).​
