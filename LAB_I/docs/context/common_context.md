# Общий контекст лабораторных работ по Explainable AI (XAI)

Этот документ объединяет данные и код из четырех блокнотов для обеспечения полного контекста ИИ-агенту. Работа охватывает интерпретацию моделей регрессии (прогноз цен на авто) и классификации (распознавание цифр).

## 1\. Регрессия: Набор данных Cars Moldova

### Описание данных

Прогноз стоимости автомобиля (Price) на основе характеристик:

- **Категориальные:** Make, Model, Style, Fuel_type, Transmission.
- **Числовые:** Year, Distance (пробег), Engine_capacity(cm3).

### Предобработка (Base scikit-learn)

- **Очистка:** Удаление дубликатов, обработка выбросов (фильтрация по квантилям для цены и пробега).
- **Кодирование:** - LabelEncoder для признаков с большим количеством категорий (Make, Model).
  - pd.get_dummies или аналоги для Style, Fuel_type, Transmission.
- **Масштабирование:** StandardScaler для корректной работы линейных моделей.
- **Метрики:** MAE, MSE, R^2.

## 2\. Глобальная интерпретируемость (Baseline)

В базовых работах используются:

- **Feature Importance (Random Forest):** Оценка важности признаков на основе уменьшения примеси (Gini/Entropy).
- **Коэффициенты линейной регрессии:** Показывают вес и направление влияния каждого признака (после масштабирования).

## 3\. Локальная интерпретируемость: LIME

**Инструмент:** lime.lime_tabular.LimeTabularExplainer

**Ключевые особенности:**

- Строит локальную аппроксимацию вокруг конкретного примера.
- В регрессии показывает, какие условия (например, Year > 2010) «тянут» цену вверх или вниз.
- **Пример кода:**

explainer = lime_tabular.LimeTabularExplainer(  
X_train.values,  
feature_names=X_train.columns,  
mode='regression'  
)  
exp = explainer.explain_instance(X_test.iloc\[0\], model.predict)

## 4\. Продвинутая интерпретируемость: SHAP (Tabular)

**Инструмент:** shap.TreeExplainer (для RF/XGBoost) или shap.KernelExplainer.

**Типы визуализаций в работах:**

- **Summary Plot (Beeswarm):** Показывает распределение влияния каждого признака на весь набор данных.
- **Waterfall Plot:** Детальный разбор одного предсказания (от базового значения до финального прогноза).
- **Dependence Plot:** Анализ зависимости целевой переменной от одного признака с учетом взаимодействия с другими.

## 5\. XAI для изображений: MNIST + SHAP

### Описание задачи

Классификация рукописных цифр (28x28 пикселей).

### Метод интерпретации

**Инструмент:** shap.Explainer (обычно с использованием DeepExplainer или PartitionExplainer).

**Логика:**

- Алгоритм определяет, какие группы пикселей внесли наибольший вклад в пользу конкретного класса (например, цифры «7»).
- **Визуализация:** - Красные зоны - пиксели, повышающие вероятность класса.
  - Синие зоны - пиксели, понижающие вероятность.
  - Позволяет понять, на какие «черточки» цифры смотрит модель.

## 6\. Технический стек и зависимости

- **Core:** scikit-learn, pandas, numpy, matplotlib, seaborn.
- **XAI:** lime, shap.
- **Models:** RandomForestRegressor, LinearRegression, DecisionTree.

## 7\. Чек-лист для агента при выполнении заданий

- **Этап 1 (EDA):** Проверь распределение Price и корреляцию признаков.
- **Этап 2 (Моделирование):** Обучи модель и зафиксируй R^2.
- **Этап 3 (LIME):** Выбери "типичный" и "аномальный" автомобиль. Объясни разницу.
- **Этап 4 (SHAP Tabular):** Сравни топ-признаки SHAP с Feature Importance леса. Есть ли расхождения?
- **Этап 5 (SHAP Images):** Для ошибочно классифицированных цифр MNIST выведи карту важности пикселей, чтобы понять причину ошибки.
