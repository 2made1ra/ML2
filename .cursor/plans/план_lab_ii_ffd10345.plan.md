## Стратегия выполнения по разделам ноутбука
1. **Подготовка окружения и воспроизводимость**
   - Добавить импорты (`torch`, `torchvision`, `numpy`, `matplotlib`, `seaborn`, `sklearn.metrics`).
   - Зафиксировать seed для `random`, `numpy`, `torch` (и для MPS/CUDA при наличии).
   - Ввести единый конфиг (`device`, `batch_size`, `epochs`, `lr`, `num_workers`) и утилиты логирования метрик.

2. **Данные и предобработка**
   - Загрузить FashionMNIST через `torchvision.datasets`.
   - Разделить train на train/val (например, 90/10) с фиксированным seed.
   - Построить базовые визуализации: по 1-2 примера на класс и проверка баланса классов.
   - Подготовить transform-пайплайны: базовый (ToTensor+Normalize) и расширенный (с умеренной аугментацией для экспериментов).

3. **Базовая модель (baseline)**
   - Реализовать простую CNN (например, 2 сверточных блока + FC) как отправную точку.
   - Обучить фиксированное число эпох с `Adam`/`CrossEntropyLoss`.
   - Зафиксировать baseline-метрики: train/val loss, val accuracy, test accuracy.
   - Добавить матрицу ошибок и краткую интерпретацию сильных/слабых классов.

4. **Улучшение модели (4-6 экспериментов)**
   - Провести серию изолированных изменений (по 1 ключевой гипотезе за эксперимент):
     - усиление архитектуры (больше каналов/слоёв);
     - `BatchNorm`;
     - `Dropout`;
     - scheduler для LR (`StepLR` или `CosineAnnealingLR`);
     - аккуратная аугментация (горизонтальный флип не всегда уместен для одежды; приоритет — малые affine-преобразования).
   - Для каждого эксперимента сохранять одинаковый формат логов: параметры, лучшая val accuracy, test accuracy, время обучения.
   - Выбрать лучшую конфигурацию по val и подтвердить на test.

5. **Анализ вклада улучшений (абляция)**
   - Свести результаты в компактную таблицу: baseline vs эксперименты vs best model.
   - Отдельно выделить:
     - что дало наибольший прирост;
     - что почти не повлияло/ухудшило;
     - гипотезы, почему так произошло.
   - Оставить только информативные графики/результаты без визуального шума.

6. **Анализ ошибок итоговой модели**
   - Построить confusion matrix для лучшей модели.
   - Выделить 2-3 наиболее часто путаемых пары классов.
   - Показать минимум 3 характерные ошибки (картинка, true label, pred label, короткий комментарий причины ошибки).

7. **Финальные выводы и проверка оформления**
   - Заполнить выводы по целям задания: достигнутая точность, ключевые факторы роста качества, ограничения.
   - Проверить, что в ноутбуке нет падающих ячеек, есть поясняющий текст и логичная структура разделов.
   - Подготовить ноутбук к сдаче: последовательный прогон сверху вниз.
